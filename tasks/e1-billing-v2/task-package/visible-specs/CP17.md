# CP17: Deterministic audit replay

The replay_hash query folds the event log and returns a deterministic hash of the resulting state. Replaying the same events always reproduces the same hash; every piece of state must be derivable from events alone.

## Invariants introduced (cumulative from now on)

- I-REPLAY: replaying the event log reproduces the state hash exactly.

## Scenarios

#### Scenario: Replay is stable
- **GIVEN** any event list
- **WHEN** replay_hash is queried twice
- **THEN** both hashes are identical

#### Scenario: Hash covers the full state
- **GIVEN** two event lists differing in one applied event
- **WHEN** replay_hash is queried for each
- **THEN** the hashes differ

## Queries introduced

- `replay_hash` with fields: `kind`

## Worked examples

Each example is `evaluate(events, query) -> expected`, verbatim. These exact cases are the checkpoint's acceptance examples.

```json
{
  "check_id": "cp17-hash-basic",
  "args": [
    [
      {
        "event_id": "S1-create",
        "type": "subscription_created",
        "at": "2026-01-01T00:00:00.000Z",
        "subscription_id": "S1",
        "customer_id": "C-S1",
        "plan_id": "basic",
        "plan_price_cents": 3000,
        "period_start": "2026-01-01",
        "period_end": "2026-01-31"
      }
    ],
    {
      "kind": "replay_hash"
    }
  ],
  "expected": "53a4238f"
}
```
