# CP04: Partial shipments

Warehouses ship multi-line orders in waves. A paid order with at least one shipped line and at least one unshipped line is now reported as `partially_shipped` everywhere a status is shown; such orders previously appeared as processing. In lifecycle order, partially_shipped sits between processing and shipped.

## Scenarios

#### Scenario: First wave shipped
- **GIVEN** a paid two-line order with exactly one line shipped
- **WHEN** the order status is queried
- **THEN** the status is partially_shipped

#### Scenario: Final wave completes the order
- **GIVEN** a partially shipped order
- **WHEN** its remaining lines ship
- **THEN** the status is shipped

## Worked examples

Each example is `evaluate(events, query) -> expected`, verbatim. These exact cases are the checkpoint's acceptance examples.

```json
{
  "check_id": "cp04-status-partial",
  "args": [
    [
      {
        "event_id": "P1-create-101",
        "type": "order_created",
        "at": "2026-01-01T00:00:00.000Z",
        "order_id": "P1",
        "lines": [
          {
            "line_id": "P1-L1",
            "amount_cents": 3000
          },
          {
            "line_id": "P1-L2",
            "amount_cents": 4000
          }
        ]
      },
      {
        "event_id": "P1-pay-102",
        "type": "payment_received",
        "at": "2026-01-01T00:00:00.000Z",
        "order_id": "P1"
      },
      {
        "event_id": "P1-ship-103",
        "type": "line_shipped",
        "at": "2026-01-01T00:00:00.000Z",
        "order_id": "P1",
        "line_id": "P1-L1"
      }
    ],
    {
      "kind": "order_status",
      "order_id": "P1"
    }
  ],
  "expected": {
    "order_id": "P1",
    "status": "partially_shipped"
  }
}
```
